# MP10 · Landing Page Deployment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deploy `apps/landing/` to Cloudflare Pages via GitHub Actions so every push to `main` auto-deploys the landing page to a free `*.pages.dev` URL.

**Architecture:** GitHub Actions workflow builds the Astro site and deploys via `cloudflare/wrangler-action`. Secrets stored in GitHub repo settings. No code changes needed — landing page is already complete.

**Tech Stack:** Astro 5, Cloudflare Pages, GitHub Actions, `cloudflare/wrangler-action@v3`

---

## Prerequisites (manual — do these before running any task)

These steps happen outside the codebase. Complete all four before starting Task 1.

**Step A — Create Cloudflare account**
- Go to https://dash.cloudflare.com/sign-up
- Sign up with your email (free tier, no credit card)

**Step B — Create the Pages project**
- In CF dashboard: Workers & Pages → Create → Pages → "Create directly"
- Project name: `meridian-landing`
- Skip the Git connection (CI/CD will handle deploys)
- Click "Create project"

**Step C — Get API token + Account ID**
- Account ID: CF dashboard sidebar → bottom-left, copy the Account ID string (32 hex chars)
- API token: My Profile (top-right) → API Tokens → Create Token → use template "Edit Cloudflare Pages" → Continue to summary → Create Token
- Copy the token — it's shown once only

**Step D — Add GitHub secrets**
- Go to your GitHub repo → Settings → Secrets and variables → Actions → New repository secret
- Add `CLOUDFLARE_API_TOKEN` = the token from Step C
- Add `CLOUDFLARE_ACCOUNT_ID` = the Account ID from Step C

## Task 1: Create GitHub Actions workflow

**Files:**
- Create: `.github/workflows/deploy-landing.yml`

- [ ] **Step 1: Create the workflows directory**

```bash
mkdir -p .github/workflows
```

- [ ] **Step 2: Write the workflow file**

Create `.github/workflows/deploy-landing.yml` with this exact content:

```yaml
name: Deploy Landing

on:
  push:
    branches: [main]
    paths:
      - 'apps/landing/**'
      - '.github/workflows/deploy-landing.yml'
  pull_request:
    paths:
      - 'apps/landing/**'
      - '.github/workflows/deploy-landing.yml'

jobs:
  deploy:
    runs-on: ubuntu-latest
    name: Deploy to Cloudflare Pages
    permissions:
      contents: read
      deployments: write
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: apps/landing/package-lock.json

      - name: Install dependencies
        run: npm ci
        working-directory: apps/landing

      - name: Build
        run: npm run build
        working-directory: apps/landing

      - name: Deploy to Cloudflare Pages
        uses: cloudflare/wrangler-action@v3
        with:
          apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
          command: pages deploy dist --project-name meridian-landing
          workingDirectory: apps/landing
```

- [ ] **Step 3: Verify workflow file is valid YAML**

```bash
node -e "require('fs').readFileSync('.github/workflows/deploy-landing.yml', 'utf8'); console.log('valid')"
```

Expected output: `valid`

- [ ] **Step 4: Commit**

```bash
git add .github/workflows/deploy-landing.yml
git commit -m "ci(mp10): GitHub Actions deploy landing to Cloudflare Pages"
```

## Troubleshooting

**Workflow doesn't trigger on push:**
- Confirm the pushed branch is `main` (not `master`)
- Confirm changed files match the `paths` filter (`apps/landing/**`)

**Deploy step fails with "project not found":**
- Confirm the Pages project name in CF dashboard is exactly `meridian-landing`
- Confirm `CLOUDFLARE_ACCOUNT_ID` matches the account where the project was created

**Deploy step fails with "authentication error":**
- Regenerate the API token in CF dashboard and update the GitHub secret
- Confirm the token has `Cloudflare Pages: Edit` permission

**Build fails with missing dependency:**
- Run `npm ci && npm run build` locally inside `apps/landing/` to reproduce
- Fix locally, commit, push
